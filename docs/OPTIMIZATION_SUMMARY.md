# 性能优化与数据提供商调研 - 执行摘要

## 核心结论

### 1. 当前最大瓶颈：指标重复计算
```
每只股票的指标被重复计算3次/策略:
  - filter(): 1次
  - calculate_dimensions(): 1次
  - build_match_reasons(): 1次

2000只股票 × 8策略 × 3次 = 48,000次重复计算

优化后: 2000只股票 × 1次 = 2,000次计算

节省: 15-20分钟
```

### 2. Polygon.io免费套餐能力

| 功能 | 限制 | 价值 |
|------|------|------|
| **Snapshot** (全市场快照) | 1 call获取全部2000只 | ✅ **高** - 推荐集成 |
| **技术指标** | 5 calls/min = 300/hour | ⚠️ **有限** - 只能用于Top候选 |
| **历史数据** | 多次调用 | ❌ **不足** - 免费套餐不够 |

**关键发现**:
- Snapshot API 1次调用获取全市场 → 替代2000次yfinance调用
- 技术指标API速率限制 → 只能用于30-50只候选股票验证

### 3. 推荐优化方案

#### Phase 1（立即实施）- 节省20分钟
1. ✅ **指标计算缓存**（最高优先级）
   - 每只股票只计算一次技术指标
   - 所有策略共享计算结果

2. ✅ **批量数据库插入**
   - 当前: 436,000次单独INSERT
   - 优化: executemany() → 10-50倍提升

3. ✅ **RS统一预计算**
   - 避免VCP-EP和Momentum重复计算RS

#### Phase 2（下周）- 额外节省5分钟
4. ✅ **集成Polygon Snapshot**
   - 1次API调用获取全市场当日数据
   - 验证yfinance数据一致性

5. **yfinance批量下载**
   - 使用 `yf.download(symbols, threads=True)`
   - 5-10倍速度提升

### 4. 预期性能提升

| 场景 | 当前 | 优化后 | 提升 |
|------|------|--------|------|
| 全量扫描 | 70-80分钟 | **15-20分钟** | 4-5倍 |
| 增量扫描 | 20-25分钟 | **5-8分钟** | 3-4倍 |

### 5. 数据提供商策略

```
推荐架构: yfinance为主 + Polygon Snapshot为辅

yfinance (免费):
  ├── 历史OHLCV数据 ✓
  ├── 本地指标计算 ✓
  └── 增量更新 ✓

Polygon.io (免费 - 5 calls/min):
  ├── Snapshot (全市场) ✓ - 每日1次
  ├── Top候选指标验证 - 可选
  └── 不用于全量指标获取 ✗ (速率限制)
```

### 6. 为什么不完全迁移到Polygon.io？

**免费套餐限制**:
- 需要8,000 calls/天获取全量技术指标
- 免费套餐仅7,200 calls/天
- 加上Snapshot和其他调用，**免费套餐不够用**

**成本考量**:
- Polygon.io付费套餐: $49/月起
- 本地计算优化后已足够快（15-20分钟）
- ROI不高，除非需要实时数据
